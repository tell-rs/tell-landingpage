Tell now isolates each workspace into its own database. Events, sessions, and logs from one workspace can never appear in another — not in queries, not in dashboards, not anywhere.

## Workspace isolation

Each workspace gets a dedicated database. Data boundaries are enforced at the query level — there's no shared table to accidentally leak across.

## Ingestion keys — create and rotate instantly

Manage the keys your SDKs and collectors use to send data. Create a key, name it, and start ingesting immediately — no server restart required.

- **Create from the CLI** — `tell apikeys new mobile-prod` generates a key and shows it once. It's live in the pipeline the moment it's created.
- **List and delete** — `tell apikeys` shows all keys by name and prefix. `tell apikeys delete` revokes a key instantly.
- **API access** — admins can manage keys programmatically via REST endpoints for automation and CI/CD workflows.
